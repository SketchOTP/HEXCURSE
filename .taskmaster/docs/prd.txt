# Overview
HexCurse is a software project workspace that operates under strict Cursor agent governance. The repository holds the governance kit (Cursor rules, directives, session logs), system documentation, and will grow to hold application code once the product stack is confirmed. Primary users are the project owner and AI implementation agents following AGENTS.md.

# Core Features
- Governed AI sessions: MCP-driven workflow (Taskmaster, memory, Serena, context7, repomix, gitmcp, GitHub, sequential-thinking).
- Living documentation: ARCHITECTURE.md, DIRECTIVES.md, SESSION_LOG.md kept in sync with Taskmaster tasks.json.
- Task graph: Dependency-aware directives parsed from this PRD via Taskmaster.

# User Experience
- Human confirms scope once per directive; agent executes tooling, branches, PRs, and logging automatically per AGENTS.md.

# Logical Dependency Chain
1. Finalize governance files and MCP usage rules (foundation for all future work).
2. Parse PRD → tasks.json; sync DIRECTIVES.md human-readable layer.
3. Record setup session in SESSION_LOG.md.
4. Only after Phase A is complete: begin application code per future directive.

# Risks and Mitigations
- Missing API keys for Taskmaster parse-prd: use `.env` from `.env.example`; document fallback manual task list.
- MCP servers inactive: verify global `mcp.json`; document in ARCHITECTURE.
- Scope creep: sacred constraint — one directive per session; backlog for new ideas.
