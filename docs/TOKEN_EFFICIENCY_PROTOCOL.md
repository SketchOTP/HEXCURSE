# Token Efficiency Measurement Protocol — HexCurse

## Why this matters

HexCurse's core promise is that AI sessions get more accurate with less token waste over time. This protocol makes that claim measurable.

## What to measure

For each session, record in **SESSION_LOG.md**:

- **Input tokens:** tokens consumed by context loading (rules, docs, memory, tool results)
- **Output tokens:** tokens generated by the agent
- **Tool calls:** number of MCP tool invocations
- **Redundant reads:** number of times the agent read a file it had already read this session
- **Task outcome:** COMPLETE / PARTIAL / BLOCKED

## How to estimate token counts

Cursor does not expose a token counter directly. Estimate using:

1. Cursor's context usage indicator (visible in agent mode — percentage of context used)
2. Rule of thumb: 1 token ≈ 4 characters of English text
3. For precise measurement: copy session content to a tokenizer (e.g. tiktoken) and count

## Baseline measurement (Session 1)

Record the following for the first session in a new repo:

- Time to complete SESSION START ritual (minutes)
- Estimated input tokens for context loading
- Number of tool calls in SESSION START
- Number of files read that were never used in the session

## Efficiency signals (improving trend)

The system is working if, by Session 5+:

- Time for SESSION START ritual has decreased
- Memory MCP returns relevant context (reducing fresh reads)
- Fewer files read that are never used
- Fewer redundant tool calls (PAMPA finds existing skills instead of rediscovering)

## Efficiency signals (stagnating)

Investigate if, by Session 5+:

- Every session still reads the same large files from scratch
- Memory MCP returns no relevant memories
- No skills have been promoted
- Token counts are not decreasing

## SESSION_LOG.md token tracking format

Add to each SESSION_LOG entry:

```
**Token estimate:** ~{N} input / ~{N} output
**Tool calls:** {N} total ({N} MCP, {N} file reads, {N} redundant)
**Context efficiency:** {percentage of context used at peak}
```

## Reporting cadence

After every 5 sessions, run **`--learning-rollup`** and add a **`## Token Trend`** section to **ROLLING_CONTEXT.md** comparing average token usage across the last 5 sessions vs the first 5.
